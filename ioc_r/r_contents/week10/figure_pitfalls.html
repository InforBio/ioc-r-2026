Les pièges de visualisation

week10

Une bonne figure scientifique doit :

  • répondre à une question claire
  • être facile à lire
  • représenter les données sans biais
  • éviter les interprétations trompeuses

Beaucoup de figures posent problème à cause de :

  • mauvais type de graphique
  • trop d’informations
  • couleurs mal choisies
  • axes trompeurs

Ordre et lisibilité

Des données non ordonnées rendent la lecture difficile.

Bonne pratique : ordonner les catégories pour faciliter la comparaison.

Exemples :

  • trier par valeur
  • trier par ordre biologique
  • trier par temps

Axe Y : commencer à 0 ou non ?

Les barplots doivent commencer à 0. Sinon la différence est exagérée visuellement.

library(tidyverse)

df <- data.frame(
  group = c("A", "B"),
  value = c(52, 55)
)

ggplot(df, aes(group, value)) +
  geom_col()
ggplot(df, aes(group, value)) +
  geom_col() +
  coord_cartesian(ylim = c(50, 56))

Surcharge d’information

Symptômes :

  • trop de groupes
  • trop de couleurs
  • trop de panels
  • trop d’annotations
  • trop de variables dans un même plot

Règle simple : une figure doit répondre à une seule question scientifique.

Example : les line plots deviennent illisibles avec > 5 groupes.

Solutions : facets par groupe, mettre en évidence quelques groupes importants, griser les autres groupes, labeliser directement les lignes, etc.

Mauvais graphiques : pie charts

Les pie charts sont difficiles à lire :

  • comparaison d’angles difficile
  • comparaison d’aires difficile

Alternatives :

  • barplot
  • lollipop plot
  • treemap

Example:

cell_types <- tibble(
  cell_type = c(
    "T cells",
    "B cells",
    "NK cells",
    "Monocytes",
    "Dendritic cells",
    "Epithelial",
    "Fibroblasts"
  ),
  proportion = c(31, 21, 8, 15, 7, 12, 6)
)

ggplot(cell_types, aes(x = "", y = proportion, fill = cell_type)) +
  geom_col(width = 1) +
  coord_polar(theta = "y") +
  theme_void()

Barplot :

  • comparaison immédiate
  • tri possible
  • facile à lire
ggplot(cell_types,
       aes(x = reorder(cell_type, proportion),
           y = proportion,
           fill = cell_type)) +
  geom_col() +
  coord_flip() +
  theme_minimal() +
  labs(x = "", y = "Percentage")

Lollipop plot :

  • plus léger visuellement
  • très lisible
ggplot(cell_types,
       aes(x = reorder(cell_type, proportion),
           y = proportion)) +
  geom_segment(aes(xend = cell_type,
                   y = 0,
                   yend = proportion),
               color = "grey70") +
  geom_point(size = 4, color = "steelblue") +
  coord_flip() +
  theme_minimal() +
  labs(x = "", y = "Percentage")

Treemap :

  • représentation intuitive de la surface
  • utile quand il y a beaucoup de catégories
  • Bon pour visualiser des proportions relatives
library(treemapify)
ggplot(cell_types,
       aes(area = proportion,
           fill = cell_type,
           label = cell_type)) +
  geom_treemap() +
  geom_treemap_text(colour = "white", place = "centre") +
  theme(legend.position = "none")

Histogrammes

Le bin width influence fortement l’interprétation.

Un mauvais choix peut :

  • cacher une distribution bimodale
  • créer de faux motifs

Bonne pratique : tester plusieurs valeurs.

geom_histogram(binwidth = ...)
geom_histogram(bins = ...)

Barres d’erreur

Les barplots avec mean ± error bar peuvent cacher :

  • outliers
  • distribution bimodale
  • tailles d’échantillons différentes

De plus, les barres d’erreur peuvent représenter :

  • SD (écart-type, mesure de la dispersion des données)
  • SE (erreur-type, mesure de la précision de la moyenne)
  • CI (intervalle de confiance, l’intervalle probable de la vraie moyenne)

C’est souvent ambigu pour le lecteur.

Alternatives :

  • boxplot
  • violin plot
  • nuage de points

Mauvaises couleurs

Bonnes pratiques :

  • utiliser ≤ 6–7 couleurs
  • garder la même couleur pour la même variable
  • préférer les couleurs sobres pour le contexte
  • utiliser la couleur pour souligner l’information importante

Attention :

  • éviter les gradients pour des catégories
  • préférer des palettes color-blind friendly
  • garder un bon contraste

Solution simple :

scale_color_viridis_d()
scale_fill_viridis_d()

# ou
library(RColorBrewer)
scale_fill_brewer(palette = "Set2")

Final checklist

Avant de publier une figure :

  • Le graphique répond-il à une question claire ?
  • Les axes sont-ils honnêtes ?
  • Peut-on voir les données ?
  • Les couleurs sont lisibles ?
  • La figure est-elle compréhensible sans explication orale ?
